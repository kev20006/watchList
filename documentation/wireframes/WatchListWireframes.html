Discussion:

As described the app will consist of a main viewport in which list items will be rendered as cards. All menus will be hidden by default to keep the interface as minimal as possible to remove clutter.

Changes form the design:

The original plan was to have the cards expand which selected to display more options. Unfortunately in practice this didn’t render very well and either caused multiple cards to resize at the same time or a lot of white space being created.

In the live version cards show a the movie or tv show poster, and when hovered or selected the information and controls slide up from the bottom of the card.

 

Adding a new item

Drop down menu:

Clicking the + icon will expand the dropdown to show all the options.

The search Interface:

Selecting an item will render the search interface in a modal. Click a search result will open the item preview

Item Preview:

Item preview will be rendered in the same modal window as the search results.

Item Preview Actions:

Clicking the add button on the item preview would add the item to the list

Clicking the back button on the item preview will return to the search results

The process of adding an item has remained largely unchanged from the original design. However the amount of information and the layout of the preview has changed somewhat.

Draw Menu

The drawer menu has remained largely unchanged since the initial designs. However, throughout implementation some more functionality was added which has been included in the drawer menu on the live version.

Additionally, in practice I felt it looks better when the drawer menu utilised 100% of the available height, so now slides over the navbar and all content below that.
